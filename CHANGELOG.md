# 更新履歴

## [v2.17.1]
### 変更
 - 起動時に音声認識エンジンを Soniox へ強制切替するコードを削除し、利用者が選んだエンジン設定を尊重するようにしました。
 - 音声認識エンジンの既定値を Soniox から `ブラウザ内蔵 (Web Speech API)` に戻しました。新規利用者は無料エンジンで起動します。
 - エンジン選択肢の表記を `Web Speech API (無料)` から `ブラウザ内蔵 (無料)` に変更し、利用者にとって意味が分かりやすい表現にしました。
 - 翻訳テストカードのタイトルを「翻訳テスト」から「翻訳」に短縮しました。
 - 翻訳結果欄が長文で枠を押し広げてしまう不具合を修正しました。カード高さを 250px に固定し、結果欄が縦スクロールするようにしました。
 - Deepgram / Soniox 選択時に「オフライン認識パック」行（ボタンとラベル）を非表示にし、不要な項目を隠すようにしました。

## [v2.17.0]
### 変更
 - `その他` パネル（オプション）のレイアウトを 3 列カード式から 2 列テーブル式へ刷新しました。左側 40% に設定リスト（字幕表示・音声認識・OBS連携・システム の 4 セクション）、右側 60% に固定高さ 250px の翻訳テストカードを配置し、設定の分類と視認性を改善しました。
 - 各セクション内の設定項目を機能ごとに整理し直しました。「字幕パネル最大化」を「システム」へ、「字幕単一行表示」「Overflow」を「字幕表示」へ、「自動停止 (5h)」を「音声認識」セクションの先頭へ移動しました。
 - 設定行内の select / カラーピッカー / Overflow の radio-box を全て幅 160px に揃え、視覚的に縦のラインで揃うようにしました。
 - 自動停止無効時の `⚠️ OFF` 警告バッジを、セクションタイトル位置から該当行の末尾に移動し、警告対象を直感的に分かるようにしました。
 - Overflow（翻訳字幕行数制限）の選択肢を行の右側に縦並びで配置するように変更し、ラベルの「(Overflow)」表記を削除しました。
 - 翻訳テストカードを左右 50:50 の水平分割（入力 | 結果）に変更し、入力と翻訳結果を同時に確認できるようにしました。
 - スクロールバーをオプションパネル全体から左側設定リストのみに集約し、Webkit / Firefox 両対応のカスタムスクロールバー (棕色半透明) を適用しました。

## [v2.16.5]
### 追加
 - 翻訳バックエンド `/translate` で `FORCE_STOP_CLIENTS=true` 環境変数が設定されている場合、レスポンスに `stop:true` を付与し、クライアント側の録音を自動停止できるようにしました。使用者がページを閉じ忘れて予算を消費し続ける状況での緊急停止用の防護機構です。
 - 平常時は無効（環境変数未設定）。発動が必要な時のみ Docker 等で `FORCE_STOP_CLIENTS=true` を追加して再起動します。

## [v2.16.4]

 - 翻訳テストでは Ray Mode の入力側・翻訳後フィルタを通常の音声フローと同じように適用します。
### 変更

## [v2.16.0]
### 追加

 - Soniox 用のトレース機能を追加しました。DevTools コンソールで `__sonioxDownloadTrace()` を実行すると、最近 100 件の token / flush / endpoint イベントを JSON ファイルとしてダウンロードできます (切句問題のデバッグ用)。
### 変更

### 修正
 - Soniox 側の endpoint 検出とクライアント側の換気タイマー (1秒) が競合し、同じ文が 2〜3 回翻訳に送られる問題を修正しました。クライアント側の換気タイマーを完全に廃止し、切句は Soniox の endpoint detection に一本化しました。

 - ブラウザ判定で大文字 / 小文字の違いにより誤判定される問題を修正しました。
 - 使っていないコードを一部削除しました。
 - 自動再起動エラー時の stop パラメータを削除しました。
   理論上、この使い方は onend イベントによる再起動と衝突する可能性があります。ただし、多くの場合はオブジェクト自体がまだ起動中の状態でエラーが出るため、削除後に再起動エラーが出続ける可能性もあります。
   実際にどうなるかはしばらく様子を見る必要があります。現状では onend イベントが出たときだけ再起動するため、onend が出た時点で停止済みです。仮に再起動に失敗しても停止状態のままなので、エラーが出続けることはないはずです。

 - 停止ボタンのイベントで使う stop パラメータの処理を abort に変更しました。stop は停止時に結果を返しますが、abort は停止時の結果も拒否します。
   テストでは stop が音声認識を止められないことがあり、その後の起動時に「起動中」のエラーが出て再開できない場合がありました。安定性に問題が出るため、音声認識の停止には abort を使うようにしました。abort は強制停止後に onerror を送信し、その後 onend に流れます。理論上はこちらのほうが安定しますが、実際の挙動は引き続き確認します。
